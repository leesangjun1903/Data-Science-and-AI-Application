# AI 엔지니어링 스택: 미래형 AI 애플리케이션 개발의 완전 가이드

## AI 엔지니어링 스택의 3계층 구조

AI 애플리케이션을 성공적으로 구축하려면 **세 가지 핵심 계층**을 이해해야 합니다. 이 구조는 현대 AI 개발의 기초이며, 각 계층이 유기적으로 연결되어 완성된 AI 시스템을 만들어냅니다.[1]

### **1계층: 애플리케이션 개발 (Application Layer)**

애플리케이션 개발은 **사용자와 직접 상호작용하는 최상위 계층**입니다. 이 계층에서는 다음과 같은 핵심 요소들이 중요합니다:[1]

**프롬프트 엔지니어링과 컨텍스트 구성**
- 모델에게 적절한 지시사항과 컨텍스트를 제공하는 기술[1]
- Gemini Ultra의 경우 프롬프트 기법 적용으로 MMLU 점수가 83.7%에서 90.04%로 향상[1]
- 단순한 명령을 넘어서 메모리 관리 시스템까지 포함하는 포괄적 접근

**평가 시스템**
- 위험 요소 감소와 개선 방향 탐색을 위한 핵심 과정[1]
- 파운데이션 모델의 개방형 특성으로 인해 전통적 ML보다 훨씬 복잡
- 정답이 무한히 다양할 수 있는 작업에서의 평가 난이도 증가

**AI 인터페이스**
- 웹/데스크톱/모바일 앱 (Streamlit, Gradio, Plotly Dash)[1]
- 브라우저 확장 프로그램과 챗봇 통합
- 플러그인 및 애드온 형태의 API 연동
- 음성 기반 및 AR/VR 인터페이스까지 확장

### **2계층: 모델 개발 (Model Development)**

모델 개발은 **전통적 ML 엔지니어링과 가장 밀접한 계층**으로, 다음 핵심 영역들로 구성됩니다:[1]

**모델링과 학습**
- 구조 설계부터 학습, 파인튜닝까지의 전 과정
- TensorFlow, Transformers, PyTorch 등 프레임워크 활용[1]
- 다양한 ML 알고리즘과 신경망 아키텍처에 대한 전문 지식 필요

**학습 단계별 구분**
- **사전 학습(Pre-training)**: 무작위 초기화 상태에서 시작하는 처음부터 학습[1]
- **파인튜닝(Fine-tuning)**: 기존 학습된 모델 기반 추가 학습[1]
- **후속 학습(Post-training)**: 모델 개발자가 특정 목적으로 진행하는 추가 학습[1]

**데이터셋 엔지니어링**
- 데이터 수집, 생성, 라벨링의 모든 과정 포함[1]
- 정형 데이터에서 비정형 데이터로의 패러다임 전환
- 중복 제거, 토큰화, 컨텍스트 검색, 품질 관리가 핵심

**추론 최적화**
- 모델의 속도와 비용 효율성 개선[1]
- 파운데이션 모델의 순차적 생성 방식으로 인한 지연 시간 문제 해결
- 100ms 수준의 응답 속도 구현이라는 도전적 목표

### **3계층: 인프라 (Infrastructure)**

인프라는 **모든 AI 시스템의 기반**이 되는 최하위 계층입니다:[1]

- 모델 배포 및 서비스 운영 시스템
- 데이터와 컴퓨팅 자원의 효율적 관리
- 시스템 모니터링 및 성능 최적화 도구
- GPU 클러스터 관리 (10대에서 1,000대 규모까지)[1]

## AI 엔지니어링 vs ML 엔지니어링: 3가지 핵심 차이점

### **1. 모델 개발 접근 방식의 전환**

**전통적 ML 엔지니어링**: 직접 모델 학습과 개발에 집중[1]
**AI 엔지니어링**: 기존 파운데이션 모델의 효과적 활용에 집중[1]

이러한 변화로 인해 AI 엔지니어들은 모델 자체 개발보다는 **모델 맞춤화(Model Adaptation)**에 더 많은 시간을 투자합니다.

### **2. 컴퓨팅 자원 요구량의 급격한 증가**

파운데이션 모델은 전통적 ML 모델 대비 다음과 같은 특징을 보입니다:[1]
- **훨씬 큰 모델 크기**와 더 높은 컴퓨팅 자원 소모
- **더 높은 레이턴시**와 추론 비용
- **대규모 GPU 클러스터** 운영의 필요성

### **3. 개방형 결과와 평가의 복잡성**

**전통적 ML**: 스팸 분류같은 닫힌 형태(close-ended) 작업[1]
**AI 엔지니어링**: 창의적 답변 생성같은 열린 형태(open-ended) 작업[1]

이로 인해 **평가 시스템의 설계**가 훨씬 더 중요하고 복잡한 과제가 되었습니다.

## AI 엔지니어링과 풀스택 개발의 융합

### **기술 스택의 다양화**

전통적인 파이썬 중심에서 자바스크립트 생태계로 확장:[1]
- LangChain.js
- Transformers.js  
- OpenAI Node 라이브러리
- Vercel AI SDK

### **개발 접근 방식의 변화**

**전통적 ML 엔지니어링**: 데이터 수집 → 모델 학습 → 제품 개발[1]
**현대 AI 엔지니어링**: 제품 프로토타입 → 검증 후 데이터/모델 투자[1]

이러한 변화로 **빠른 프로토타이핑과 반복 개선**이 가능해졌으며, AI 엔지니어들이 제품 결정 과정에 더 깊이 관여하게 되었습니다.[1]

## 모델 맞춤화 전략: 프롬프트 기반 vs 파인튜닝

### **프롬프트 기반 기법**
- **가중치 변경 없이** 입력만으로 모델 동작 유도[1]
- 비교적 쉬운 시작, 적은 데이터 요구량
- 빠른 실험과 모델 비교 가능
- 복잡한 작업에서는 한계 존재

### **파인튜닝 기법**
- **모델 가중치 직접 수정**을 통한 맞춤화[1]
- 더 복잡하지만 품질, 속도, 비용 면에서 큰 개선 효과
- 새로운 작업 적용 시 필수적 접근법

## 생태계 성장 동향과 미래 전망

2024년 GitHub 분석 결과에 따르면, Stable Diffusion과 ChatGPT 등장 후 2023년에 AI 관련 도구가 급격히 증가했습니다. 특히 **애플리케이션과 애플리케이션 개발 분야**에서 가장 큰 성장을 보였으며, 이는 AI 기술의 민주화를 의미합니다.[1]

AI 엔지니어링은 여전히 **ML 엔지니어링의 기본 원칙들**을 계승하면서도, 파운데이션 모델 시대에 맞는 새로운 혁신을 더해가고 있습니다. 이 분야는 아직 시작 단계에 불과하며, 앞으로도 무궁무진한 발전 가능성을 품고 있는 미래형 기술 영역입니다.[1]
